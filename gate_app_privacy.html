Privacy Policy for Barda37 Gate App

Last updated: 2025-11-25

1. Introduction

Barda37 Gate App ("the App") allows authorized residents/users to authenticate and remotely control designated gates. This policy explains what data we collect, how we use it, and your choices. By using the App you agree to this policy.

2. Data We Collect

  • Account / Credentials: Username (entered by user) for authentication and personalization
  • Authentication Tokens: JWT/session token stored securely to maintain login
  • Permissions: Gate access permissions to determine allowed actions
  • Remember Me Flag: Boolean preference to skip biometric on subsequent launches if user opts in
  • Biometrics (Local): Fingerprint/face verification status (local only, not transmitted)
  • Advertising Data: AdMob may collect device identifiers & usage data for ads
  • Device Data: IP address, device model, OS version (via AdMob/Play services)
  • Audit Log: When a gate is triggered, the server stores a log entry including timestamp, username, and IP address. Logs are used for security and accountability and are automatically rotated (maximum size 1MB).

We do not knowingly collect: precise location, contacts, photos, or unrelated personal data. The App does not request access to contacts, photos, microphone, camera, precise location, or any sensitive device permissions.

3. How We Use Data

  • Authenticate users and authorize gate control actions
  • Maintain persistent login if "Remember Me" is enabled
  • Show relevant ads via AdMob
  • Improve reliability, security, and fraud detection

4. Biometrics

Biometric authentication is performed locally using device secure hardware/OS APIs. Biometric data never leaves the device and is not stored by the App.

Biometric authentication is optional and is controlled entirely by the deviceâ€™s operating system. The App receives only a success/fail result.

5. Remember Me Behavior

If enabled, the App stores a flag allowing future launches to bypass biometric re-authentication until logout or disabled. For enhanced security on shared devices, keep biometrics enabled.

6. Advertising (AdMob)

The App integrates Google AdMob. AdMob may collect device identifiers, approximate location, and usage metrics to serve and measure ads. For details, see Google's Privacy Policy and AdMob Help.

7. Data Storage & Security

  • Tokens and sensitive session data are stored using secure storage mechanisms
  • Network requests use HTTPS
  • Only authorized gates appear for control

8. Data Sharing

We do not sell your personal data. Minimal data may be shared with service providers strictly to operate functionality (e.g., backend server for gate commands; Google AdMob for displaying ads). Legal requests may require disclosure.

9. Retention

Session tokens persist until logout or token expiry. Gate permissions refresh on login. Removing the App or clearing storage may remove local data.

10. Your Choices

  • Disable "Remember Me" to enforce biometric each session
  • Logout to clear stored token and preferences
  • Manage ad personalization through your Google account or device settings

Account & Data Deletion
